# CO2 Emission Prediction: A Multi-Model Machine Learning Benchmark

This repository contains the code accompanying the paper **[paper title]**.  
It provides a complete pipeline for benchmarking regression models on provincial CO2 emission data in China, covering data splitting, feature selection, cross-validated training, and bootstrap uncertainty estimation.

---

## Repository Structure

```
.
├── data/
│   ├── raw/                  # Place merged9.5.csv here (not tracked by git)
│   └── processed/            # Generated by preprocess.py
├── src/
│   ├── data.py               # CSV I/O utilities
│   ├── features.py           # Feature selection (MI, RF, Pearson)
│   ├── models.py             # Model registry and per-model training functions
│   ├── training.py           # Cross-validated grid search + task runner

├── config.py                 # Global configuration (paths, hyperparameters)

└── requirements.txt
```

## Data

Place the raw dataset at:

```
data/raw/merged9.5.csv
```

The dataset contains panel data for Chinese provinces/cities with the following variable groups:

| Group | Variables |
|---|---|
| Identifiers | `prov_id`, `uid`, `year` |
| Climate | `dew`, `temp`, `evap`, `wind`, `press`, `solar`, `precip`, `pm25` |
| Socioeconomic | `pop`, `gdp`, `area_crop`, `area_forest`, `area_urban`, `val_ind`, `export`, `import`, `val_pri`, `val_sec`, `val_ter`, `inc_urban`, `inc_rural`, `pass_vol`, `cars` |
| Energy | `coal_raw`, `coal_clean`, `briquettes`, `coke`, `gas_coke`, `oil_crude`, `gasoline`, `kerosene`, `diesel`, `oil_fuel`, `lpg`, `gas_ref`, `gas_nat`, `heat`, `elec` |
| Target | `co2_ds` (CO2 emissions, downscaled) |

## Workflow

### Step 1 — Data Splitting

```bash
python preprocess.py
```

This generates two sets of train/test splits under `data/processed/`:

| Split | Train | Test | Criterion |
|---|---|---|---|
| **Spatial** (`s`) | 80 % of administrative units (uid) | 20 % of units | Random by uid (seed=42) |
| **Temporal** (`t`) | Years 2001–2016 | Years 2017–2020 | Chronological |

### Step 2 — Training

```bash
python main.py
```

The pipeline iterates over all combinations of `SCALES × FEATURE_METHODS × SELECT_K_LIST` defined in `config.py`, and trains every model in `MODELS`.

Results are saved to `data/predict_ds/{scale}_{method}_{k}/{model_name}/`.

---

## Configuration

All experiment settings are in `config.py`:

| Parameter | Default | Description |
|---|---|---|
| `SCALES` | `['t']` | Data splits to use (`'s'` spatial, `'t'` temporal) |
| `FEATURE_METHODS` | `['mi']` | Feature selection method (`'mi'`, `'rf'`, `'pearson'`, `None`) |
| `SELECT_K_LIST` | `[20]` | Number of features to select |
| `TARGET_COL` | `'co2_ds'` | Regression target |
| `K_FOLDS` | `5` | Cross-validation folds |
| `N_BOOTSTRAP` | `20` | Bootstrap iterations for uncertainty |
| `DO_UNCERTAINTY` | `False` | Enable/disable uncertainty estimation |
| `SELECTED_FEATURES` | (list) | Variable whitelist applied before feature selection |

To **enable or disable a model**, comment/uncomment the corresponding entry in the `MODELS` dict in `src/models.py`.

## Models

| Category | Models |
|---|---|
| Linear | `linear`, `ridge`, `bayesianridge`, `sgd`, `huber`, `glm` |
| Instance-based / Kernel | `knn`, `svr` |
| Neural networks | `mlp`, `elm` |
| Tree ensembles | `rf`, `gbdt`, `adaboost`, `bagging`, `histgb` |
| Boosting libraries | `xgb` (XGBoost), `lgb` (LightGBM), `catboost` |

Each model is tuned via 5-fold grid search over a predefined hyperparameter grid (see `src/models.py`).

---

## Output Files

For each model, the following CSV files are produced:

| File | Contents |
|---|---|
| `*_train_summary.csv` | Best CV MSE and wall-clock training time |
| `*_cv_fold_metrics.csv` | MSE / RMSE / MAE / R² for each CV fold |
| `*_gridsearch_results.csv` | Full GridSearchCV result table |
| `*_test_metrics.csv` | Test-set MSE / RMSE / MAE / R² and prediction time |
| `*_uncertainty.csv` | Bootstrap mean prediction with 95 % CI (if enabled) |
| `*_bootstrap_metrics.csv` | Per-iteration bootstrap metrics (if enabled) |

---

## Feature Selection Methods

| Method | Description |
|---|---|
| `mi` | Mutual Information (`mutual_info_regression`) |
| `rf` | Random Forest feature importance |
| `pearson` | Absolute Pearson correlation with the target |
| `None` | No selection; all whitelist features are used |
